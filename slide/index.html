AMQP Message Structure

Key Concepts

1. Publisher and Consumer

Publisher is a program that sends messages.

Consumer is a program that mostly waits to receive messages.

2. Queues

A queue is the name for a mailbox.

It can store as many messages as you like ‒ it's essentially an infinite buffer.

3. Exchanges

Producers can only send messages to an exchange. An exchange on one side it receives messages from producers and the other side it pushes them to queues.

4. Exchanges Types

  • Direct
  • Fanout
  • Topic
  • Headers

5. Binding

Relationship between Exchange and Queue

6. Routing Key

routing_key is an extra parameter for binding


channel.queue_bind(exchange=exchange_name,
         queue=queue_name,
         routing_key='black')

7. Virtual Hosts

Virtual Host is a concept to run multiple hosts in a server.

Reliability

RabbitMQ offers a variety of features to let you trade off performance with reliability, including persistence, delivery acknowledgements, publisher confirms, and high availability.

Flexible Routing

Messages are routed through exchanges before arriving at queues. RabbitMQ features several built-in exchange types for typical routing logic. For more complex routing you can bind exchanges together or even write your own exchange type as a plugin.

Clustering

Several RabbitMQ servers on a local network can be clustered together, forming a single logical broker.

Federation

Exchange Types

There are 4 Exchange Types when we bind with Queues

1. Direct

routes messages with a routing key equal to the same routing key

2. Fanout

routes messages to all bound queues

3. Topic

routes messages to queues whose routing key matches all, or a portion of a routing key

asterisk (“*”) to match a word in a specific position of the routing key

hash (“#”) to match zero or more words

4. Headers

routes messages based upon a matching of message headers to the expected headers specified by the binding queue

When the "x-match" argument is set to "any", just one matching header value is sufficient

When the "x-match" argument is set to "all", all the values must match

Message properties

The AMQP protocol predefines a set of 14 properties that are rarely used, with the exception of the following:

  • persistent: Marks a message as persistent (with a value of true) or transient (false). You may remember this property from the second tutorial.
  • content_type: Used to describe the mime-type of the encoding. For example for the often used JSON encoding it is a good practice to set this property to: application/json.
  • reply_to: Commonly used to name a callback queue.
  • correlation_id: Useful to correlate RPC responses with requests.

Persistance

3 conditions for persisting messages in Rabbit MQ

  1. Mark the exchange “durable:true”
  2. Mark the queue “durable:true”
  3. Set the message’s “delivery mode” to a value of 2 (persistent:true)

* RabbitMQ server restart will remove the data without these options

ACK vs NACK

Consumer Prefetch

Consumer Prefetch

limit the number of unacknowledged messages on a channel (or connection) when consuming (aka "prefetch count")

Example (java)


Channel channel = ...;
Consumer consumer1 = ...;
Consumer consumer2 = ...;
channel.basicQos(10, false); // Per consumer limit
channel.basicQos(15, true);  // Per channel limit
channel.basicConsume("my-queue1", false, consumer1);
channel.basicConsume("my-queue2", false, consumer2);

These two consumers will only ever have 15 unacknowledged messages between them, with a maximum of 10 messages for each consumer.

Error Handling

DLX(Dead Letter Exchange)

Messages from a queue can be 'dead-lettered'

  • The message is rejected (basic.reject or basic.nack) with requeue=false
  • The TTL for the message expires
  • The queue length limit is exceeded.

a DLX can be defined by clients using the queue's arguments, or in the server using policies

Examples

send.js


var amqp = require('amqplib/callback_api');

amqp.connect('amqp://localhost', function(err, conn) {
  conn.createChannel(function(err, ch) {
    var q = 'hello';
    var msg = 'Hello World!';

    ch.assertQueue(q, {durable: false});
    ch.sendToQueue(q, new Buffer(msg));
    console.log(" [x] Sent %s", msg);
  });
  setTimeout(function() { conn.close(); process.exit(0) }, 500);
});

Examples

receive.js


var amqp = require('amqplib/callback_api');

amqp.connect('amqp://localhost', function(err, conn) {
  conn.createChannel(function(err, ch) {
    var q = 'hello';

    ch.assertQueue(q, {durable: false});
    console.log(" [*] Waiting for messages in %s. To exit press CTRL+C", q);
    ch.consume(q, function(msg) {
      console.log(" [x] Received %s", msg.content.toString());
    }, {noAck: true});
  });
});
